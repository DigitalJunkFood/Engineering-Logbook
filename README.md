## About

This repository serves as my personal engineering logbook—a chronological record of my journey from learning fundamental concepts to designing complete engineering systems.

Rather than documenting only finished projects, this repository captures the entire engineering process, including experimentation, debugging, design revisions, research, failures, and the lessons learned from each stage of development.

The goal is to demonstrate not only **what I build**, but **how I think**.

---

## Purpose

Engineering is an iterative process.

Every failed test, wiring mistake, software bug, and redesign contributes to a better understanding of the system.

This logbook documents those experiences in real time, preserving the reasoning behind design decisions instead of only showcasing polished final results.

---

# Engineering Philosophy

One of the most valuable engineering tools is documentation.

A successful experiment confirms a hypothesis.

A failed experiment reveals information that can improve the next design.

Both outcomes are worth documenting.

This repository intentionally includes unsuccessful tests, debugging sessions, and design revisions because they represent the engineering process just as much as successful builds.
